The **FMLS Application** is designed for collecting and storing survey data in areas with intermittent or no internet connectivity. It allows users to save survey responses locally and upload the data to a remote server once an internet connection is available. This feature is crucial for government surveys and field data collection in remote areas.

- ### Key Features
  - **Local storage**: stores survey data in a local database when there is no internet connection.
  - **Data synchronization**: uploads the stored data to a remote server once a network connection is available.
  - **Ideal for field data collection**: supports surveys in areas with poor or no connectivity, particularly for government and large-scale surveys.

- ### My Contributions
  - Developed a feature to **upload data** from the **local database** to the **remote server**.
  - Created a screen to display the **number of items uploaded** and those **pending synchronization**.
  - Implemented various **UI modifications** based on client requirements to improve usability.

- **Technologies Used**: Kotlin, Room Database, Retrofit, and Android SDK.
